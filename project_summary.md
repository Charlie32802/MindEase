# MindEase - Project Summary

MindEase is an integrated digital mental health platform developed specifically for young adults in Surigao City. Built on a **hybrid care model**, it seamlessly combines a non-clinical AI Companion (for initial symptom triage and psychoeducation) with secure Professional Teleconsultation Services.

To ensure genuine accessibility, the project employs a **dual-tier operational model**: it remains entirely cost-free for individual users to eliminate financial barriers, while designed for institutional licensing (e.g., educational organizations) to sustain server and AI hosting costs.

## 1. App Structure
- **`accounts`**: Manages the custom `User` model (via `AbstractBaseUser` and `PermissionsMixin`), authentication (login/registration with email verification), and profile data. Handles three main roles — User, Counselor, and Admin — each with role-specific fields (e.g., age, gender, and emergency contacts for users; specializations stored as JSON, institution details, license number, years of experience, bio, and uploaded credential files for counselors). Provides separate registration forms for users (with privacy policy acceptance) and counselors (with file upload support for professional IDs and degree certificates). Also serves legal compliance pages: Privacy Policy, Terms of Service, and Professional Code of Ethics.
- **`core`**: The heart of the platform. Manages user and counselor dashboards, counselor availability (including vacation mode with date ranges), appointment booking with race-condition-safe locking, a "Find a Counselor" page (lazy-loaded with pagination), in-app user and counselor profile update pages (with live analytics like average rating, review count, and profile views), patient records (paginated and searchable), a past sessions page (with sorting by latest, oldest, A-Z, or Z-A), counselor feedback and analytics (ratings, reviews, best review, profile view tracking), anonymous system feedback submission via a "Get Support" page (with an `is_resolved` tracking flag), and live video sessions (via Agora RTC). Also includes a clipboard-ready weekly report generator and a clear-week-schedule endpoint for counselors.
- **`ai_companion`**: Provides a 24/7 AI chatbot powered by **Groq's LLaMA 3.3 70B Versatile** model. Features include a 3-pass NLP pipeline (main response → semantic counselor-need evaluator → contextual transition generator), principle-based prompt engineering, dynamic injection of live counselor data, and a zero-token local regex intent router for instant UI guidance. Chat history is persisted via `Conversation` and `Message` models, with full database-backed history regeneration (including real-time slot availability re-checks on historical counselor cards) on every chat open.
- **`admin_panel`**: A comprehensive custom `AdminSite` dashboard for administrators to verify counselors, manage users, track site-wide activities, manage wellness tips (full CRUD), view system and counselor feedback (via paginated tab views), and browse a Philippines news feed (NewsAPI / RSS / GNews fallback chain with 15-minute caching). Features an admin-only email-based login with access enforcement, role-based dynamic user creation forms, a sidebar context processor that injects live counts (total users, counselors, pending approvals, feedback) into every admin page, and activity logging for admin-performed actions (user/counselor additions, approvals, rejections).
- **`email_notifications`**: Handles outbound HTML emails (counselor approval/rejection notifications, user email verification activation links) via Gmail SMTP. Uses `EmailMultiAlternatives` with HTML template rendering and plaintext fallback for all transactional emails.
- **`config`**: The Django project configuration package. Contains `settings.py` (database, caching, email, Groq API, news API keys, Redis channel layer, logging, and ASGI configuration), `asgi.py` (ASGI application with `ProtocolTypeRouter` routing HTTP and WebSocket traffic through `AuthMiddlewareStack`), `urls.py` (root URL configuration including all app routes and custom admin site), and `wsgi.py`. A `run_server.bat` script is provided for quick local server startup.

## 2. User Flows
### User (Patient) Flow
1. **Onboarding**: Registers with full name, email, age, gender, emergency contact details (name, phone, relationship), and privacy policy acceptance. Receives a verification email with an activation link and activates their account before first login. Can review the Privacy Policy, Terms of Service, and Professional Code of Ethics during registration.
2. **Dashboard**: Accesses the User Dashboard to view today's appointments (with Join Session and Cancel buttons), upcoming sessions, admin-curated wellness tips, and Quick Settings shortcuts (Chat With AI, Book Session, Past Sessions, Get Support). If a completed session has no feedback yet, a feedback prompt is automatically triggered.
3. **AI Support & Discovery**: Chats with the MindEase AI widget (available on every page). If the user shares a problem, the AI's 3-pass pipeline evaluates severity, dynamically selects the best-matching verified counselor, generates a natural transition sentence, and renders a "Book Session" card directly in the chat — with a SweetAlert2-powered slot picker showing real-time slot capacity. If the recommended counselor has no available slots, the widget starts a 30-second polling mechanism and shows a persistent notification bubble and updates the card to an active "Book Session" button when a slot opens. Common navigational queries (e.g., "How do I book?", "How to cancel?") are intercepted by a local regex intent engine that instantly types out pre-written UI guides without consuming any AI tokens.
4. **Manual Booking**: Can also browse the "Find a Counselor" page (lazy-loaded with pagination) to filter by name or specialization, view ratings/reviews/profile views/vacation dates, pick an available time slot via the SweetAlert2 slot picker, and book — with both client-side (JavaScript time-check) and server-side (slot end-time comparison) booking deadline enforcement. Users are limited to one booking per day.
5. **Session**: When the scheduled time arrives and the counselor starts the session, the user receives a real-time WebSocket notification and joins the Agora video call directly from the in-app session page — no external app or login required. During the session, both parties can exchange text messages and emoji reactions via an in-session WebSocket chat.
6. **Post-Session**: After the counselor ends the session, the user is prompted to submit feedback with an optional 1–5 star rating and a required text message. Feedback is one-time only per completed appointment.
7. **Profile & History**: Can update personal profile information (name, age, gender, emergency contacts). Can view all past sessions with sorting options (latest, oldest, A–Z, Z–A), including session status, duration, counselor specializations, and any submitted feedback.
8. **Get Support**: Can submit anonymous support messages to the platform administrators through a dedicated "Get Support" page (name field is optional).

### Counselor Flow
1. **Onboarding**: Registers with professional credentials (first and last name, institution details, institution email, license number, years of experience, specializations, bio, and privacy policy acceptance). Uploads professional ID and degree certificate files. Is redirected to a "Registration Pending" page upon successful registration.
2. **Verification**: Waits for an Admin to verify their account. Receives an HTML approval or rejection email notification.
3. **Availability**: Logs in and uses the "Manage Availability" page to set specific dates, time slots, and slot capacities. Can copy last week's schedule, clear the current week, or set vacation mode (date ranges that block bookings and display on counselor cards). Can view and delete upcoming vacation entries (grouped by consecutive dates).
4. **Management**: Uses the Counselor Dashboard to view today's appointments, this week's schedule, weekly stats (patients, sessions, average rating), and recent feedback. Can generate a clipboard-ready weekly report (completed sessions, missed sessions, patient names) and clear this week's pending appointments.
5. **Session**: Clicks "Start Session" to create an Agora RTC channel (named `mindease{appointment_id}`), which sends a real-time WebSocket notification to the patient. Enters the in-app session page with access to the patient's past clinical notes and an in-session text/emoji chat. Ends the session by writing counselor notes; session duration is automatically computed in total seconds from the patient's join timestamp and displayed in a human-readable `X mins Y secs` format.
6. **Profile & Records**: Can update professional profile (name, specializations, institution, license, experience, bio, credential files) through a dedicated profile page showing live analytics (average rating, total reviews, profile views). Can browse all patient records (paginated, searchable by patient name) including session dates, durations, statuses, and clinical notes.

### Admin Flow
1. **Dashboard Overview**: Logs into the custom Admin Panel (admin-only email-based login that blocks non-admin roles) to see high-level metrics (total users, counselors, pending counselors, admins) and paginated recent activities. The sidebar displays live counters for users, counselors, pending approvals, and total feedback.
2. **Verification**: Reviews pending counselor applications, examines their uploaded credentials (professional ID, degree certificate, institution details, license number), and approves or rejects them — triggering an automatic HTML email notification to the counselor and logging the action in recent activities.
3. **User & Counselor Management**: Views and manages user/counselor lists with the ability to add (via a role-based dynamic creation form that enforces role-specific required fields), edit, or deactivate accounts. Shows verification status (Verified, Pending Review, or Rejected) for counselors.
4. **Wellness Tips**: Creates, edits, and deletes wellness tips (title and description) that appear on user dashboards.
5. **Feedback Monitoring**: Reviews both system feedback (anonymous support messages from the "Get Support" page) and counselor feedback (user-submitted session reviews with ratings) via paginated tab views with a tab switcher.
6. **Activity Tracking**: Monitors all platform activities (registrations, logins, logouts, email verifications, bookings, cancellations, first chatbot uses, feedback submissions, admin-performed user additions, counselor approvals/rejections) through a paginated recent activity feed with FontAwesome icon indicators and human-readable relative timestamps ("Just now", "5 minutes ago", "Yesterday").
7. **News Feed**: Browses a Philippines news feed aggregated from NewsAPI, RSS feeds (PhilStar, Rappler, CNN Philippines), and GNews — with automatic 15-minute caching via Django's local memory cache.

## 3. Technical Highlights
- **Backend Stack**: Django (Python) running on **Daphne (ASGI)**. Utilizes Django Channels with a **Redis Channel Layer** for real-time WebSocket notifications (session started, session ended, appointment booked/cancelled, availability removed) across multiple parallel workers. Two WebSocket consumers: `SessionNotificationConsumer` (per-user channel for platform-wide notifications) and `SessionChatConsumer` (per-session channel for in-session messaging).
- **Database**: **MySQL** (`mindease_db`) with `utf8mb4` charset and `STRICT_TRANS_TABLES` mode. Models span five apps: `User` (accounts), `Appointment`, `Availability`, `CounselorFeedback`, `CounselorProfileView`, `SystemFeedback` (core), `Conversation`, `Message` (ai_companion), `RecentActivity`, `WellnessTip` (admin_panel).
- **Concurrency & Data Integrity**: Implements robust database locking (`transaction.atomic` and `select_for_update()`) to eliminate race conditions, specifically during high-contention events like multi-user slot booking. Includes both client-side (JavaScript time-check in the SweetAlert2 picker) and server-side (slot end-time comparison against current time) booking deadline enforcement. Users are restricted to one booking per counselor per day.
- **Frontend Stack**: Vanilla HTML/CSS/JS with **FontAwesome** icons throughout the interface. Uses `marked.js` (bundled locally via `node_modules`) for rendering Markdown from the AI with GFM and line-break support, and `SweetAlert2` for interactive modals (slot picker with real-time capacity, booking confirmations, cancellation prompts, feedback forms). AI chat responses are rendered with a **character-by-character typewriter animation** that properly handles HTML tags and entities.
- **AI Integration & 3-Pass NLP Pipeline**: Custom principle-based prompt engineering with the **Groq API** (**LLaMA 3.3 70B Versatile**). The system prompt is built on five core principles — Proportionality, Conversational Authenticity, Structural Restraint, Unsolicited Restraint, and Guided Professional Intervention — plus a Platform Orientation directive and an Uncompromising Crisis Protocol with Surigao City emergency hotlines. The three sequential API calls per user message are: **(1)** the main conversational response (temperature 0.7, max 800 tokens) with dynamically injected live counselor data (names, specializations, experience, bio, ratings), **(2)** a secondary semantic triage evaluator (`temperature=0.0`, JSON response format) that determines if the user's distress level warrants human intervention and selects the single best-matching counselor, and **(3)** a transition sentence generator (temperature 0.5, max 150 tokens) that naturally bridges the AI's response to the rendered counselor booking card.
- **Zero-Token Intent Routing**: `widget.js` features a local regex intent engine that intercepts five categories of common navigational queries — joining sessions, booking appointments, cancelling bookings, viewing past sessions, and contacting support. Matched queries instantly trigger a typewriter-animated, pre-written UI guide and asynchronously save the interaction to the database (preserving it in chat history), completely bypassing the 70B LLM to conserve API tokens and reduce latency.
- **Dynamic Asynchronous Polling**: Implements a highly optimized 30-second frontend polling mechanism (`setInterval` at 30,000ms) to check for newly opened counselor slots without requiring page reloads. Results are pushed to a sequential notification queue that generates persistent, reload-safe chat bubbles (saved to the database and restored on history load) and universally updates all relevant UI counselor cards from "No available slots" to active "Book Session" buttons.
- **Persistent UX State**: Utilizes `localStorage` combined with database history regeneration to maintain critical UX states — including unread notification badges (pulsing red dots via `ai_has_unread`), last-recommended counselor data (`ai_last_counselors_{sessionId}`), and session IDs (`ai_session_id`) — across full page reloads. On chat open, the system reloads full message history from the database and re-checks each historical counselor card's current slot availability in real-time, ensuring users never miss when a recommended counselor becomes available.
- **Dynamic Dashboard DOM Update**: When a user books a session from within the AI chat widget while on the dashboard page, the system dynamically re-fetches the dashboard HTML and surgically updates the "Today's Appointments" and "Upcoming Appointments" cards without a full page reload.
- **Video Calling**: Uses **Agora RTC** for secure, in-app video calls. Tokens are generated server-side using a custom HMAC-SHA256 `AccessToken` builder (`core/agora_token.py`) with publisher privileges (join channel, publish audio/video/data streams). Channels are named `mindease{appointment_id}` and tokens expire after 1 hour.
- **In-Session Communication**: Beyond video, each active session has a dedicated **WebSocket channel** (`SessionChatConsumer`) enabling real-time text messaging and emoji reactions between counselor and patient during the video call. When the counselor ends the session, a `session_ended` WebSocket event is broadcast to redirect the patient.
- **Email System**: Gmail SMTP (SSL, port 465) for transactional emails including user account activation links (with token-based verification via `default_token_generator`), counselor approval notifications, and counselor rejection notifications — all using HTML email templates with `EmailMultiAlternatives` (HTML + plaintext fallback).
- **Counselor Feedback & Analytics**: A complete review system where patients submit one-time post-session feedback with optional 1–5 star ratings and a required text message. Per-counselor analytics include aggregated average ratings, total review counts, daily-unique profile view tracking (SHA-256 hashed via session key or IP address, with `unique_together` constraint per counselor per day), and featured "best review" (most recent 5-star) display on counselor cards.
- **Activity Tracking System**: A centralized `RecentActivity` model tracks 12 distinct activity types (counselor approved/rejected, user/counselor/admin added, user registered, user login, email verified, appointment booked/cancelled, first chatbot use, feedback sent) with FontAwesome icon mapping, color coding, and human-readable relative timestamps.
- **Admin Sidebar Context Processor**: A custom Django context processor (`admin_panel/context_processors.py`) injects live sidebar counts (total users, verified counselors, pending approvals, total feedback) into every admin template, ensuring the navigation always reflects current platform state.
- **Logging & Monitoring**: Configurable logging with both file (`admin_news.log`) and console handlers for the admin panel's news feed module, using Django's built-in logging framework.

## 4. Scope, Limitations & Methodology
- **Methodology**: Systematically developed using an iterative approach that synthesizes **Design Thinking** (for user-centeredness), **Lean Startup** (for MVP validation), and **Agile Scrum** (for adaptive development).
- **Platform Scope**: The system is exclusively a web-based application (no mobile app version) and currently supports English-language interactions.
- **Target Demographic**: Designed for young adults experiencing generalized stress, anxiety, and mild depression. It is strictly for non-severe cases, featuring a mandatory human-led referral protocol for complex or high-risk issues, with Surigao City emergency hotlines embedded in the AI's crisis protocol.
- **In-Session Clinical Context**: Features automated patient record retrieval, rendering past clinical notes directly on the counselor's screen during active video sessions. Session duration is computed in total seconds from the patient's WebSocket join timestamp and stored for reporting.
- **System Feedback**: Users can submit anonymous support messages via the "Get Support" page, which are tracked (with an `is_resolved` status flag) and reviewed by administrators in the custom admin panel's feedback tab.
- **Legal Compliance**: Provides dedicated pages for Privacy Policy, Terms of Service, and Professional Code of Ethics, accessible during user/counselor registration and from the platform interface. Privacy policy acceptance is mandatory for both user and counselor registration.
